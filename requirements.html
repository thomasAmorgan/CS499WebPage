Requirements

TENTATIVE

General Requirements

  • The game will be played through an Internet Browser - Chrome, Firefox, Internet Explorer/Edge (potentially) and other compatible Browsers over the Internet.
  • The game will be played through the use of a mouse or other pointing device.
  • The first stage of the game will be as a "hotseat" game - multiple persons take turns using a single computer to play the game. The first stretch goal (under Stretch Goals) is to have the game playable across multiple browsers.
  • For all elements of the game not specifically mentioned, assume that the requirements follow directly what is outlined in the game rules (eg, rules for how many hexes a particular Bot can move, combat values for Bots, card effects, etc.)

User Interface - General Requirements

  • A two-dimensional hexagonal grid play space must be visible.
  • Users must be able to understand which elements are on which hexes; eg, Bots, Power Disks, Force Fields and Destroyed Hexes.
  • A player turn order indicator must be accessible.
  • An action point indicator must be accessible.
  • An interface for viewing and playing cards at appropriate times must be available.
  • Players must be able to use the interface to interact with the game as specified under User Interface - Setting up and User Interface - General Gameplay.

User Interface - Setting Up

  • Players must be able to validly place a combined total of 20 hex grids distributed as evenly as possible.
  • Players must receive three random starting cards.
  • Players must be able to place their starting pieces (one of each type of Bot) on any valid location.
  • Power Disks, one for each player, must be placed on the game board in random locations.

User Interface - General Gameplay

  • Initiative Phase - turn order is randomly determined and displayed for each round.
  • A player, on his or her turn, must be able to do the following:
    • Before all else, have the option to create a Power Disk on a random hex.
    • Use a total of 3 action points (AP) to perform any combination of the following actions:
      • Destroy a random hex - 1 AP
      • Warp a random grid - 1 AP
      • Draw a card - 1 AP
      • Rotate a grid - 1 AP
      • Convert any or all Power Disks - 1 AP
      • Warp their Converter Bot to the center of a random grid - 1 AP
      • Move and engage in combat for any or all Bots - 2 AP
      • Play any or all cards - 2 AP
      • Warp any playing piece (aside from Converters) to a random location - 2 AP
      • Double Warp (swap a selected grid with a random grid) - 2 AP
      • Swap any two selected grids - 3 AP
      • Warp friendly playing piece to any friendly or unoccupied hex - 3 AP
    • These actions are discussed in the game rules, and their implementation will be no different than described.
  • Users must be able to clearly see and understand the indicators for rotating and placing grids.

User Interface - Movement and Combat

  • Users must be able to clearly see and understand the indicators for Bot movement.
  • An interface must be provided for combat which allows for the following:
    • Before combat, the redistribution of Power Disks among allied Bots for each side (attacker first).
    • Special handling for if a force field is present in the combat hex.
    • The player with the most Bots in a particular combat (defender for ties) to choose which Bots engage each other.
    • Visible results of the random combat rolls.
    • Playing of Combat Cards before the random rolls are cast.

Game Rules - General Requirements

Rule Book
  • Combat and Movement must be implemented as specified in the game rules.
    • Players must be disallowed from making invalid moves.
  • Players must only be able to play cards at appropriate times - during combat for combat cards, and after spending two AP's to play any or all cards for special cards.
  • Players must be removed from gameplay after their Converter Bot is destroyed. This includes removing all other units which the player previously controlled. Any cards which the defeated player held must be transferred to the player which destroyed the C-Bot.

Stretch Goals

  • Internet-enabled multiplayer - Allowing players to play the game together through multiple computers connected via the Internet.
  • Alternative Game Modes - Game modes specified in the end of the game rules, selectable at the start of a game.
  • UI Facelift - Improving the quality of the UI from simply functional to enjoyable to look at and use.
  • Game Testing - Having volunteers test and play the game, offering feedback and improvements which could enhance the game's look or play.
