References

  • Honeycomb

    Honeycomb

    Honeycomb was integral in implementing our hexgrid. It provided the backbone for the hexagonal coordinate system we used. It provided functionality for manipulating and creating a customizable grid. Without it we would have been way more behind as we would have had to start from scratch. Honeycomb is under the MIT license, which allows us to use the code so long as we provide a mention of it's origin and creator.

  • PIXI.js

    PIXI.js was integral in creating the graphical interface for the hexgrid. Where Honeycomb provided the skeleton for the gird, PIXI allowed us to display it. Obviously as a game, the visuals are one of the most important parts of it. PIXI allows for the easy manipulation of graphics and allows for custom images and sprites to be used which will be useful for displaying bots.

  • Hexagonal Grids

    This website provided us with invaluable insight into understanding and implementing a hexagonal grid system. Even though we found Honeycomb this site really helped us understand much of the theory that Honeycomb ultimately implemented. This greater understanding of hexgrids made it easier for us when it came to creating the functions to manipulate gird tiles.
